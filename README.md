## 项目概述

本项目是一款网页版飞机大战游戏，采用模块化设计，代码结构清晰，便于维护和扩展。游戏包含玩家控制、敌机系统、弹幕系统、得分机制、生命值系统、难度递增等核心功能，并支持存档和设置功能。

## 技术栈

- **HTML5** - 页面结构和元素定义
- **CSS3** - 视觉样式、布局设计和动画效果
- **JavaScript ES6+** - 游戏逻辑、交互处理、碰撞检测
- **HTML5 Canvas** - 图形渲染
- **Web Audio API** - 音效生成
- **LocalStorage** - 数据持久化（设置、存档、最高分）

## 文件结构

项目包含三个核心文件：

```
plane-war/
├── index.html    # HTML页面结构
├── styles.css    # 样式表
└── game.js       # 游戏核心逻辑
```

各文件职责：

| 文件 | 职责 | 说明 |
|------|------|------|
| index.html | 页面结构 | 定义游戏容器、画布、UI界面、按钮等元素 |
| styles.css | 视觉样式 | 控制页面布局、颜色、动画、响应式设计 |
| game.js | 游戏逻辑 | 包含游戏引擎、实体类、碰撞检测、音效等 |

## 游戏玩法

### 基本规则

玩家控制一架飞机，通过发射子弹击毁敌机获得分数。随着分数增加，关卡提升，游戏难度逐渐增加。玩家有三条初始生命，被敌机或敌机子弹击中会损失生命，生命耗尽游戏结束。

### 敌机类型

游戏包含四种类型的敌机：

| 类型 | 大小 | 速度 | 血量 | 分数 | 颜色 | 特殊能力 |
|------|------|------|------|------|------|----------|
| 小型 | 小 | 快 | 1 | 100 | 橙色 | 不射击 |
| 中型 | 中 | 中 | 3 | 250 | 粉色 | 单发射击 |
| 大型 | 大 | 慢 | 6 | 500 | 红色 | 三发射击 |
| Boss | 超大 | 慢 | 20 | 2000 | 紫色 | 三发射击 |

### 难度递增

- 每1000分提升一个关卡
- 关卡越高，敌机生成速度越快
- 高关卡（3级以上）会出现中大型敌机
- 5级以上有几率出现Boss

### 道具系统

敌机被击毁时有几率掉落道具：

| 道具 | 图标 | 颜色 | 效果 |
|------|------|------|------|
| 火力提升 | P | 黄色 | 提升射击等级（最高3级） |
| 生命恢复 | + | 紫色 | 恢复一条生命（最高5条） |
| 射速加快 | S | 绿色 | 减少射击冷却时间 |

### 火力等级

玩家火力分为三级：

| 等级 | 射击方式 | 说明 |
|------|----------|------|
| 1级 | 单发子弹 | 初始火力 |
| 2级 | 双发散射 | 左右各一发 |
| 3级 | 三发散射 | 中间一发+左右各一发 |

## 游戏特性

### 核心功能

1. **玩家控制系统** - 键盘方向键或WASD控制飞机移动
2. **弹幕系统** - 玩家和敌机均可发射子弹
3. **碰撞检测** - 子弹与敌机、敌机与玩家、道具与玩家的碰撞检测
4. **得分机制** - 击毁敌机获得分数，支持最高分记录
5. **生命值系统** - 玩家有生命限制，被击中后有2秒无敌时间
6. **难度递增** - 随着关卡提升，敌机生成速度和类型增加
7. **存档功能** - 支持保存和加载游戏进度
8. **设置功能** - 可调整音效音量、背景音乐、粒子效果等

### 视觉效果

1. **霓虹风格** - 青色发光边框和按钮效果
2. **粒子系统** - 爆炸效果和飞机尾迹
3. **星空背景** - 动态闪烁的星星背景
4. **无敌闪烁** - 玩家受伤后短暂闪烁无敌
5. **血条显示** - 中大型敌机显示血量条

### 音效系统

1. **射击音效** - 子弹发射声音
2. **爆炸音效** - 敌机爆炸声音
3. **受伤音效** - 玩家受伤声音
4. **道具音效** - 拾取道具声音
5. **背景音乐** - 循环播放的游戏音乐

## 核心类说明

### 游戏引擎类

#### Game

游戏主类，负责游戏流程控制。

**主要方法：**

| 方法 | 功能 |
|------|------|
| init | 初始化游戏，绑定事件 |
| start | 开始游戏 |
| pause | 暂停游戏 |
| resume | 恢复游戏 |
| restart | 重新开始 |
| gameOver | 游戏结束 |
| update | 更新游戏状态 |
| render | 渲染游戏画面 |
| gameLoop | 游戏主循环 |
| checkCollisions | 碰撞检测 |
| spawnEnemy | 生成敌机 |
| updateHUD | 更新界面信息 |
| saveGame | 保存游戏 |
| loadGame | 加载游戏 |

### 实体类

#### GameObject

所有游戏实体的基类，提供位置、尺寸和碰撞检测功能。

**主要方法：**

| 方法 | 功能 |
|------|------|
| getBounds | 获取碰撞边界 |
| isColliding | 检测与其他实体的碰撞 |

#### Player

玩家类，继承自 GameObject。

**属性：**

| 属性 | 说明 |
|------|------|
| lives | 生命值 |
| score | 得分 |
| level | 关卡 |
| powerLevel | 火力等级 |
| fireRate | 射击冷却时间 |
| invincible | 是否无敌 |

**方法：**

| 方法 | 功能 |
|------|------|
| update | 更新玩家状态 |
| shoot | 发射子弹 |
| takeDamage | 受到伤害 |
| addScore | 添加分数 |

#### Enemy

敌机类，继承自 GameObject。

**属性：**

| 属性 | 说明 |
|------|------|
| type | 敌机类型 |
| health | 当前血量 |
| maxHealth | 最大血量 |
| scoreValue | 击杀得分 |
| movePattern | 移动模式（直线/蛇形） |

**方法：**

| 方法 | 功能 |
|------|------|
| update | 更新敌机状态 |
| shoot | 发射子弹（中大型） |
| takeDamage | 受到伤害 |

#### Bullet

子弹类，继承自 GameObject。

**属性：**

| 属性 | 说明 |
|------|------|
| vx | 水平速度 |
| vy | 垂直速度 |

#### PowerUp

道具类，继承自 GameObject。

**属性：**

| 属性 | 说明 |
|------|------|
| type | 道具类型 |

**方法：**

| 方法 | 功能 |
|------|------|
| applyEffect | 应用道具效果 |

### 系统类

#### SoundManager

音效管理器，使用 Web Audio API 生成音效。

**方法：**

| 方法 | 功能 |
|------|------|
| init | 初始化音频上下文 |
| playShoot | 播放射击音效 |
| playExplosion | 播放爆炸音效 |
| playHit | 播放受伤音效 |
| playPowerUp | 播放道具音效 |
| startBGM | 开始背景音乐 |
| stopBGM | 停止背景音乐 |
| setVolume | 设置音量 |

#### ParticleSystem

粒子系统，处理爆炸效果和尾迹。

**方法：**

| 方法 | 功能 |
|------|------|
| createExplosion | 创建爆炸效果 |
| createTrail | 创建尾迹效果 |
| update | 更新粒子状态 |
| render | 渲染粒子 |

## 开发说明

### 代码结构

游戏采用面向对象的模块化设计：

1. **配置层** - CONFIG 对象集中管理游戏参数
2. **系统层** - SoundManager、ParticleSystem 处理音效和粒子
3. **实体层** - GameObject 及其子类处理游戏对象
4. **引擎层** - Game 类控制游戏流程

### 扩展建议

1. **新增敌机类型** - 在 Enemy 类中添加新类型配置
2. **新增道具类型** - 在 PowerUp 类中添加新类型
3. **添加关卡系统** - 修改难度递增逻辑
4. **添加特效** - 在 ParticleSystem 中添加新效果
5. **添加排行榜** - 扩展 LocalStorage 存储

### 注意事项

1. 游戏使用 requestAnimationFrame 实现60FPS渲染
2. 使用 LocalStorage 存储数据，注意浏览器隐私设置
3. 音效使用 Web Audio API，部分浏览器可能需要用户交互后才能播放
4. Canvas 尺寸固定为 800x600，通过 CSS 缩放适配不同屏幕
